# iDRAC IP 설정 (전면 LCD 패널 지원 모델)

1. 초기 화면 확인: LCD 패널에 Ready 또는 서버 이름이 표시된 상태에서 [OK] 버튼을 누릅니다.
2. Setup 메뉴 진입: [▶] 방향키를 눌러 [Setup]을 찾은 뒤 [OK]를 누릅니다. (모델에 따라 View ➔ iDRAC 순서일 수 있습니다.)
3. iDRAC 선택: 방향키를 이용해 [iDRAC] 메뉴를 찾아 [OK]를 누릅니다.
4. DHCP 비활성화: IPv4 ➔ Enable ➔ Yes를 순서대로 확인한 후, DHCP 항목을 No 또는 Static으로 변경합니다.
5. 고정 IP 입력: IP 메뉴에서 [OK]를 누르면 첫 번째 숫자가 깜빡입니다. 방향키로 숫자를 맞추고 [OK]를 눌러 다음 칸으로 이동하며 IP(예: 192.168.000.120)를 완성합니다.
6. Subnet 및 Gateway 입력: IP와 동일한 방식으로 서브넷 마스크(255.255.255.000)와 게이트웨이(192.168.000.001)를 입력합니다.
7. 설정 저장: 모든 입력이 끝나면 [Set] 또는 [Apply] 화면에서 [Yes]를 눌러 저장합니다. (약 10~30초 후 iDRAC에 반영됩니다.)

# BIOS 기반 iDRAC 수동 설정 (구형 또는 LCD 없는 모델)

LCD 패널이 없거나 고장 난 경우, 모니터와 키보드를 직접 연결하여 BIOS(System Setup)에서 설정하는 것이 가장 확실한 방법입니다.

1. System Setup 진입: 서버 전원을 켜고 Dell 로고 화면 우측 상단에 옵션이 나타날 때, 키보드의 F2 키를 연타하여 System Setup으로 진입합니다.
2. iDRAC 설정 이동: 메인 화면에서 iDRAC Settings 항목을 선택합니다.
3. 네트워크 메뉴 선택: 목록에서 Network 항목을 클릭합니다.
4. IPv4 세팅 변경: 화면을 아래로 내려 IPv4 Settings 섹션을 찾습니다.
5. DHCP 해제: Enable IPv4가 설정되어 있는지 확인하고, Enable DHCP 항목을 Disabled로 변경합니다.
6. 네트워크 정보 입력: 활성화된 빈칸에 사용할 Static IP Address, Static Gateway, Static Subnet Mask를 입력합니다.
7. 저장 및 재부팅: 화면 하단 또는 좌측 상단의 Back을 누르고 Finish를 클릭한 뒤, 변경 사항 저장 팝업에서 Yes를 선택하여 서버를 재부팅합니다.

# Mac 기반 초기 접속용 네트워크 설정 (iDRAC 연결 방식별)

A. 전면 iDRAC Direct 포트 연결 (최신 모델: R630, R640 등)

iDRAC IP (서버) -> 169.254.0.3 (고정 할당됨)
Mac IP (제어 PC) -> 169.254.0.4 (Subnet: 255.255.255.128)

B. 후면 iDRAC 전용 포트 연결 (구형 모델)

서버 후면의 스패너 모양 아이콘이 있는 RJ-45 포트에 랜 케이블을 연결합니다. (iDRAC IP를 192.168.0.120으로 설정했다고 가정합니다.)

1. Mac 네트워크 설정 열기: 애플 로고 ➔ 시스템 설정(System Settings) ➔ 네트워크(Network)로 이동합니다.
2. 어댑터 세부 설정: 연결된 이더넷 어댑터를 클릭하고 세부사항(Details)... ➔ TCP/IP 탭으로 이동합니다.
3. 수동 IP 할당: 'IPv4 구성'을 DHCP 서버를 사용하여에서 수동(Manually)으로 변경합니다.
4. 네트워크 정보 입력: IP 주소에 192.168.0.100, 서브넷 마스크에 255.255.255.0을 입력하고 라우터(Gateway)는 비워둡니다.
5. 저장 및 접속 확인: 설정을 저장한 뒤, 브라우저 주소창에 https://192.168.0.120을 입력하여 초기 계정(root / calvin 또는 서비스 태그)으로 로그인이 되는지 확인합니다.

# OS 설치 및 데이터 전송용 대역 설정

하드웨어 제어(iDRAC)가 아닌, 실제 OS 배포(가상 미디어 마운트) 및 프로비저닝을 위한 물리 네트워크 설정입니다.

1. 물리 포트 연결: Mac(제어 PC)의 듀얼 랜 젠더 등을 이용해 두 번째 랜 케이블을 서버 후면의 데이터용 1번 포트(NIC 1)에 연결합니다.
2. Terraform/Ansible 변수 확인: 배포 스크립트에 사용할 IP를 정의합니다.

배포망 임시 환경 변수 예시
mac_http_ip (Mac 웹 서버 IP): 121.125.69.250
os_ip (서버 OS 타겟 IP): 121.125.69.252 (Subnet: /27)

3. Mac 어댑터 B (데이터용) 설정: Mac의 두 번째 이더넷 어댑터 설정을 열고, 위의 mac_http_ip 대역에 맞춰 IP(121.125.69.250), 서브넷, 게이트웨이를 수동으로 입력합니다. (iDRAC 제어망 대역과 절대 겹치지 않도록 주의합니다.)
4. 우선순위(Service Order) 변경: Mac 네트워크 설정 목록 아래의 ... 아이콘을 클릭하고 서비스 순서 설정(Set Service Order)을 선택합니다.
5. 순서 고정: 데이터 전송용 어댑터 B를 iDRAC 제어용 어댑터 A보다 맨 위로 드래그하여 배치한 후 저장합니다. (이 과정을 누락하면 대용량 ISO 파일 마운트 및 OS 배포 시 라우팅 꼬임으로 인한 통신 오류가 발생합니다.)
