# OS 접속(SSH)이 가능한 경우

1. SSH OS 접속
2. racadm get iDRAC.NIC.Selection (현재 네트워크 포트 상태 확인 -> 결과가 1이 아니라면 전용 포트가 비활성화)
3. racadm set iDRAC.NIC.Selection 1 (iDRAC 포트를 '뒷면 전용 포트(Dedicated)'로 강제 고정)
4. racadm racreset hard (적용을 위해 iDRAC 모듈 하드 리셋 -> 메인 OS 운영에는 영향 없음)

# RACADM이 없고 범용 Linux인 경우 -> ipmitool 사용

1. SSH OS 접속
2. ipmitool mc reset cold (iDRAC 모듈 콜드 리셋 -> 전원 강제 재부팅 효과)

# 위 명령어 내리고 약 2~3분 뒤 Mac에서 서버 뒷면 iDRAC IP(192.168.x.x)로 Ping 날려보거나 브라우저 접속.

# OS 접속(SSH)이 불가능한 경우 (비밀번호 분실/방화벽 등)

방법 A: 전면 LCD 패널을 통한 포트 모드 강제 변경

1. LCD 화면에서 [OK] 버튼을 눌러 메뉴로 진입합니다.
2. [Setup] ➔ [iDRAC] ➔ [Network] 순서로 이동합니다.
3. IP 주소 설정 메뉴들을 지나쳐서, Active NIC 또는 NIC Selection 이라는 항목을 찾습니다.
4. 이 항목이 Shared나 LOM1 등으로 되어 있다면, [OK]를 누르고 방향키를 조작해 Dedicated (전용) 로 바꿉니다.
5. [Set] 또는 [Apply]를 눌러 저장합니다.
6. 약 1~2분 뒤, Mac의 랜선을 서버 뒷면의 스패너 모양 포트에 꽂고 통신을 시도합니다.

방법 B: 전면 ⓘ (System ID) 버튼을 통한 강제 소프트 리셋

1. 서버 전면(또는 후면)에 있는 파란색 ⓘ 버튼을 손가락으로 15초 이상 꾹 누르고 있습니다.
2. 서버 내부의 팬(쿨러) 소리가 갑자기 커지거나, 버튼의 불빛이 깜빡이기 시작하면 손을 뗍니다.
3. iDRAC 칩셋이 강제로 재부팅됩니다. (메인 서비스는 다운되지 않습니다.) 약 3분 뒤 접속을 시도합니다.

방법 C: 최후의 수단 (유지보수 시간 확보 필수)

1. F2 진입: 서버 재부팅 후 F2를 눌러 BIOS ➔ iDRAC Settings에서 설정을 초기화합니다.
2. 잔류 전원 방전(Flea Power Drain): 서버 전원을 끄고, 파워 케이블 2개를 물리적으로 완전히 뽑은 뒤, 전면 전원 버튼을 30초간 눌러 메인보드 전류를 완전히 비우고 다시 켭니다. (이 방법이 iDRAC 먹통을 해결하는 가장 확실한 물리적 조치입니다.)
